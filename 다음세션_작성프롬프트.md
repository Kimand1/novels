# 다음 세션 작성 프롬프트

새로운 세션에서 아래 내용을 그대로 붙여 넣어 사용한다.

```text
E:\novel 폴더의 기존 설정 문서와 원고를 읽고 이어서 작성해줘.

먼저 반드시 확인할 문서:
- story_bible.md
- 전체줄거리.md
- 문체규칙.md
- 플롯규칙.md
- 파일생성규칙.md

그리고 기존 원고 0001.txt부터 최신화까지, 특히 0061~0080.txt를 읽고 흐름을 이어줘.

현재 진행 상태:
- 61~80화는 카일이 병참 설계자로 자리 잡는 구간.
- 71~80화에서는 상단 암살 시도, 가짜 계약서, 리디아의 공개 투자, 칠 번째 창고 사건이 진행됨.
- 0080화 마지막은 적이 카일의 선택 실패까지 계산해 두었고, 계산표 위에 발레하임 황실 인장이 남아 있는 훅으로 끝남.
- 리디아의 공개 투자로 카셀 병참선이 공개화됨.
- 세드릭은 리디아와 경쟁하지만 베르니카가 먹히지 않기 위해 협력 쪽으로 기울었음.
- 발레하임 황실, 일곱 번째 선택, 제2 후보, 이전 계산 떡밥은 배치됐지만, 아르센이 회귀자라는 사실은 아직 확정하면 안 됨. 150화 전까지는 간접 단서만 사용.

다음 작성 목표:
- 0081.txt부터 이어서 작성.
- 81~90화 구간은 story_bible 기준 "마탑의 이단적 천재"로 카일의 위치가 바뀌어야 함.
- 반드시 "마탑 공개 검증 또는 결투" 사건을 넣고, 보상은 "이리스의 협력"으로 이어지게 해줘.
- 정치 회의만으로 풀지 말고 물리적 사건, 공개 검증, 결투, 마법식 검증 같은 장면을 넣어줘.
- 각 화는 4000자 이상.
- 기존 문체 유지: 짧은 문장, 대사 중심, 설명 최소화, 마지막 3문장 안에 갈등이나 떡밥.
- 설정 문서와 떡밥 관리표를 지켜줘.
- 새 고유명사나 설정을 추가할 경우 기존 설정과 충돌하지 않게 해줘.
- 파일명은 0081.txt, 0082.txt처럼 4자리 숫자로 생성해줘.
```

필요하면 마지막 줄에 원하는 작성 범위를 추가한다.

예:

```text
0081화부터 0090화까지 써줘.
```
